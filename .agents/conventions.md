# Conventions

## Code style

- **`dotnet format` before every commit** — it applies `GoogleMapsApi/.editorconfig`. CI runs a
  format check; unformatted code fails review. (`.editorconfig` here is primarily the public-API
  analyzer config — see [`public-api.md`](public-api.md).)
- **`Nullable` is enabled** in the main library — annotate reference types, don't suppress with `!`
  unless genuinely unavoidable. `LangVersion=latest`.
- **Async-only.** Public calls are `QueryAsync`. Don't add sync wrappers or block on async
  (`.Result`/`.GetAwaiter().GetResult()`) inside the library.
- **Self-documenting code over comments.** A comment should explain *why* something non-obvious is
  done — never *what* the code does (no `// increment counter`). Prefer clear names.
- **Public members need XML doc comments** (`GenerateDocumentationFile=true`; the build would warn
  otherwise — CS1591 is suppressed but new public API should still be documented).
- Follow the established request/response shapes; see [`adding-a-new-api.md`](adding-a-new-api.md).

## Commits & changelog

- **Conventional Commits** (`feat:`, `fix:`, `chore:`, `build:`, `docs:`, `refactor:`, …). The subject
  becomes the changelog line, and the type drives the version bump.
- **Never hand-edit `CHANGELOG.md`** — it's regenerated by `release.sh` from commit messages. Put the
  user-facing wording in the commit subject. See [`build-release-ci.md`](build-release-ci.md).
- Mapping to Keep-a-Changelog sections (done automatically at release): `feat` → Added, `fix` → Fixed,
  `feat!`/breaking → Changed/Removed, etc.

## Pull requests

Follow `.github/PULL_REQUEST_TEMPLATE.md`. Before opening:
- `dotnet format` run; `dotnet build` succeeds across all TFMs; `dotnet test` passes (with a valid
  `GOOGLE_API_KEY` for integration tests).
- Public-API changes recorded in `PublicAPI.Unshipped.txt` ([`public-api.md`](public-api.md)).
- A bug fix includes a regression test ([`testing.md`](testing.md)).
- Keep PRs small and focused; link the related issue.

See [`CONTRIBUTING.md`](../CONTRIBUTING.md) for the full contributor guide.
